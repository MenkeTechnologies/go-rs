>_LANGUAGE REFERENCE

Keywords, operators, and builtins recognised by the current go-rs build. The grammar follows the Go language specification; entries below are marked Implemented when they lower to fusevm bytecode today and Planned when they are on the roadmap. Unsupported constructs are rejected at parse or compile time — never silently mis-run.

Notes

  • Automatic semicolon insertion — the lexer inserts semicolons per the Go spec, so idiomatic brace placement works without explicit terminators.
  • No garbage collector — values live on the fusevm value stack and in call-frame slots; the tracing GC of the reference toolchain is absent by design.
  • Errors, not surprises — anything not yet implemented is a parse or compile error. The reference above is the contract for what a program may use today.
